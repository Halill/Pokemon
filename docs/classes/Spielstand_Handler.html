\Spielstand_Handler

Die Klasse Spielstand_Handler beinhaltet Funktionen rund um die Spielstandverwaltung.

Ein Spielstand kann gespeichert, geladen, überschrieben und angelegt werden. Zusätzlich gibt es eine Methode check_spielstand() die prüft, ob ein Spielstand vorhanden ist. Die ranking() Methode ist für die Anzeige auf der Ranking Seite gedacht.

spielstand_speichern()

spielstand_speichern()

Diese Funktion speichert einen Spielstand bei Klick auf den Button Speichern in der game.php Zunächst wird die aktuelle Zeit in die Variable $saved_at gespeichert.

Gespeichert wird die Variable saved_at und der score.

spielstand_anlegen()

spielstand_anlegen(  $pokemon) : String

Diese Funktion wird einmalig aufgerufen, wenn der User noch keinen Spielstand besitzt.

Die Funktion wird in der intro.php aufgerufen. Die Pokemon ID wird aus der Datenbank über den Pokemonnamen ermittelt und in den Spielstand geschrieben. Daneben wird noch die User ID und die Variable saved_at in den Spielstand geschrieben. Der Score wird bei Anlage auf 0 gesetzt.

Parameters

$pokemon

Hier wird der Pokemonname des Pokemons übergeben, welchen der User ausgesucht hat

Returns

String —

Gibt über einen String aus, ob die Anlage funktioniert hat

spielstand_ueberschreiben()

spielstand_ueberschreiben(  $pokemon) : String

Diese Funktion wird aufgerufen, wenn über den Button "Spielstand anlegen" ein neuer Spielstand angelegt wird.

Datenbanktechnisch wird der vorhandene Spielstand mit dem neuen Pokemon und dem neuen Speicherdatum überschrieben.

spielstand_laden()

spielstand_laden() : \spielstand

Diese Funktionen wird bei jedem Login zunächst aufgerufen, da die Einstiegsseite die spielstand.php aufgerufen wird.

Außerdem wird diese Funktion bei jedem Aufruf der Spielstandübersicht angezeigt.

Array, in dem alle Spielstand relevanten Informationen stehen: Der Spielername, der Speicherpunkt, der Score und der Pokemonname.
